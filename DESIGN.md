# 个人账本 - 设计文档 v2

## 项目概述

一个本地 Web 服务，用于个人财务管理，包含三大模块：
1. **月度余额**：分账户记录每月余额（银行卡/支付宝/微信/信用卡/投资账户），自动汇总人民币总资产
2. **消费记录**：逐笔记录消费，支持分期和房租自动生成每月账单
3. **投资记录**：独立页面，记录投资交易，余额自动计入总资产

---

## 核心设计理念

### 花费是怎么算出来的？

用户不直接填"我花了多少钱"。而是：

```
当月花费 = 月初总余额 + 当月收入 − 月末总余额
```

月初余额就是上个月的月末余额（自动继承），用户只需要填每个账户的**月末余额**和**当月收入**，花费自动得出。

### 为什么分账户？

用户有多个账户（工行储蓄、光大储蓄、支付宝、微信、工行信用卡、富途、盈透等），直接让用户手工加总再填入，容易出错且无法追溯。分账户记录后：
- 每个账户的余额变化一目了然
- 信用卡自动显示为负数（欠款）
- 投资账户余额在投资页管理，但自动参与总资产汇总
- 外币账户自动按最新汇率换算成人民币

## 技术选型

| 层 | 选择 | 理由 |
|---|---|---|
| 后端 | Python FastAPI | 轻量、异步、自带 API 文档 (Swagger) |
| 数据库 | SQLite | 单文件、零配置、个人使用足够 |
| 前端 | Jinja2 + 原生 HTML/CSS/JS | 无需构建工具，直接渲染 |
| 图表 | Chart.js (CDN) | 资产/消费趋势可视化 |
| 汇率 | Frankfurter API | 免费、无需 API Key、覆盖主流货币 |

## 汇总计算逻辑

### 月总余额（人民币）

对某月的所有 `monthly_balances`：

```
total_balance_cny = Σ ( account_balance × exchange_rate(account.currency → CNY) )
```

- 信用卡余额通常为负数，直接加入（自动扣减）
- 投资账户的余额在投资页录入（作为 monthly_balances 的一条），同样参与汇总
- 对于没有录入余额的账户/月份，沿用上一月的余额（继承逻辑）

### 月花费

```
总余额变化 = 本月总余额 − 上月总余额
当月花费 = 本月收入 − 总余额变化
```

即：收入了多少钱，减去净增加的资产，剩下的就是花掉的。

### 投资页面独立汇总

投资页面只统计 `type=investment` 的账户，展示：
- 各投资账户余额（本币 + CNY）
- 投资总投入（所有 deposit − withdraw + buy 的总金额）
- 当前持仓市值（从 monthly_balances 取）

## 汇率自动获取

### 数据来源

使用 [Frankfurter API](https://api.frankfurter.app)，免费、无需 API Key。

```
GET https://api.frankfurter.app/latest?from=USD&to=CNY
→ {"amount": 1.0, "base": "USD", "date": "2026-05-07", "rates": {"CNY": 7.05}}
```

### 缓存策略

- 汇率每日获取一次（首次启动时拉取，之后当天用缓存）
- 存入 `exchange_rates` 表，`fetched_at` 记录获取时间
- 用户打开页面时自动刷新（如果缓存过期 > 24h）
- 手动刷新按钮：强制重新获取

```

  GET    /api/accounts                 列表

  GET    /api/incomes?year=2026&month=5

  GET    /api/expenses?year=2026&month=5&category=餐饮

  GET    /api/recurring

  GET    /api/investments?year=2026&type=buy

  GET    /api/stats/total?year=2026&month=5   当月总资产(CNY) + 花费
```

## 首次使用流程

1. 添加账户（工行储蓄、支付宝、微信、信用卡、富途...）
2. 录入每个账户的"初始余额"（开始记账那个月的余额）
3. 从第二个月开始，只需要更新有变化的账户余额
4. 系统自动：
   - 用上月余额填充本月
   - 换算外币
   - 加总所有账户
   - 根据收入推算花费
   - 为房租/分期生成当月消费记录
